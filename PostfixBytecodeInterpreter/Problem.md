**System Context**
In the execution phase of a stack-based virtual machine (e.g., JVM, WebAssembly), a compiler-generated bytecode stream is evaluated using a **LIFO operand stack**. Each token either pushes a literal onto the stack or triggers an arithmetic operation that pops operands, computes a result, and pushes it back. You are tasked with implementing this core interpretation loop.

**Problem Statement**
Given a vector of strings `tokens` representing a valid postfix bytecode sequence, simulate the stack machine’s evaluation process and return the final integer value remaining on the operand stack.

- Each token is either a signed integer or one of four operators: `+`, `-`, `*`, `/`.
- Operators consume the two most recently pushed operands. The second popped value is the left operand; the first popped value is the right operand.
- Integer division between two operands **truncates toward zero**.
- The input is guaranteed to be a syntactically valid expression with no division-by-zero cases.
- All intermediate and final results fit within a standard 32-bit signed integer.

**Examples**
**Input:** `tokens = ["2", "1", "+", "3", "*"]`
**Output:** `9`
**Explanation:** `((2 + 1) * 3) = 9`

**Input:** `tokens = ["4", "13", "5", "/", "+"]`
**Output:** `6`
**Explanation:** `4 + (13 / 5) = 4 + 2 = 6` (13/5 truncates toward zero)

**Constraints**
- `1 <= tokens.length <= 10^4`
- `tokens[i]` is either an operator (`"+", "-", "*", "/"`) or an integer in the range `[-200, 200]`.
- The expression is guaranteed to be valid, well-formed, and free of division by zero.
